## User Story

En tant que **super-admin de compte**,
je veux **consulter mon abonnement courant et l'historique de mes transactions**,
afin de **suivre mon statut de paiement, mes échéances et mes justificatifs**.

---

## Contexte

Clôt le parcours de facturation (FR-011, *Should Have*). `GET /billing/subscription` renvoie le plan, le statut et la période courante ; `GET /billing/transactions` l'historique paginé (montant, méthode, statut, date). Un job backend quotidien passe les abonnements échus à `EXPIRED` (accès modules coupé, cf. FR-007) et envoie des relances (J-7, J-1) — l'UI **reflète** ces états, elle ne les calcule pas.

---

## Périmètre

**Inclus :**
- Écran `/(app)/billing` : **carte abonnement courant** (plan, statut `ACTIVE`/`PENDING_PAYMENT`/`EXPIRED`, période courante, prochaine échéance).
- **Historique des transactions** : liste paginée (montant XOF, méthode Mobile Money/carte, statut, date) ; états loading/error/empty.
- **État `EXPIRED`** : message clair + CTA vers le catalogue de plans (FE-011) pour renouveler (paiement manuel — pas de récurrence auto en phase 1).
- **État sans abonnement** : orientation vers les plans.
- Formatage XOF + dates (i18n FR).

**Hors périmètre :**
- Création du checkout / paiement → **FE-012**.
- Relances e-mail (backend).
- Reçus/factures PDF (hors périmètre PRD phase 1).

- **Pagination** : suivre la convention backend (limit 20, max 100) ; pagination TanStack Query (page/curseur selon le contrat OpenAPI).
- **Cohérence avec FE-010** : le statut d'abonnement ici et le stepper d'onboarding lisent des sources cohérentes ; invalider `subscription` après un retour de paiement (FE-012).
- **XOF/dates** : `Intl.NumberFormat`/`Intl.DateTimeFormat` locale fr.
