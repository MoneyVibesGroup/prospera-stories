## User Story

En tant qu'**administrateur plateforme Money Vibes**,
je veux **une vue consolidée de chaque organisation (identité, statut KYC, entitlements actifs)**,
afin de **comprendre en un coup d'œil l'état d'un client et décider des actions (revue KYC, octroi de module)**.

---

## Contexte

Point d'entrée opérationnel de la console. L'admin-panel (BFF) **agrège** trois sources amont via la gateway en relayant le JWT admin : identité (`auth-service`, orgs admin STORY-027), KYC (`kyc-service`), entitlements (`platform-catalog-service` STORY-033). La vue est en **lecture** ; les actions (approve/reject KYC, grant entitlement) arrivent en AP-03/AP-05.

---

## Périmètre

**Inclus :**
- **Liste paginée des organisations** : raison sociale, pays, statut identité (actif/suspendu), statut KYC, nb d'entitlements actifs ; recherche + filtres (statut KYC, vertical).
- **Fiche détail d'une org** : agrégat identité (membres/rôles) + KYC (statut, historique) + entitlements (modules attribués, référentiel, statut).
- **Dégradation partielle** : si une source amont est indisponible, afficher les données disponibles + indicateur d'indisponibilité (jamais d'écran blanc).
- **Navigation** vers les actions (liens désactivés tant que AP-03/AP-05 non livrés).

**Hors périmètre :**
- Actions de revue KYC (AP-03) et d'entitlement (AP-05).
- Édition d'identité (reste dans l'IdP / hors console v1).
