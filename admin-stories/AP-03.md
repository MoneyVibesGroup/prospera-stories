## User Story

En tant qu'**administrateur plateforme Money Vibes**,
je veux **traiter la file des dossiers KYC en attente : consulter les pièces (RCCM/CFE) et approuver ou rejeter avec motif**,
afin d'**activer les organisations légitimes et écarter les dossiers non conformes**.

---

## Contexte

Maillon **central du dogfooding** : sans revue KYC, aucune org (y compris Money Vibes) ne passe `APPROVED`, donc aucun module ne devient utilisable. Le backend `kyc-service` expose déjà la revue admin (STORY-013 : file, détail, URLs présignées, approve/reject 404/409). Cette story livre l'UI ; elle passe par le **BFF admin-panel** (STORY-048) quand il est prêt, sinon appelle `kyc-service` directement via la gateway avec le JWT admin.

Après décision, le backend émet `kyc.status.changed` → les read-models (app cliente, dashboard) se mettent à jour (cohérence éventuelle).

---

## Périmètre

**Inclus :**
- **File de revue** : dossiers au statut `UNDER_REVIEW` (org, date de soumission, documents) ; tri/pagination.
- **Détail d'un dossier** : métadonnées + **liens de téléchargement présignés** des pièces (RCCM/CFE), affichage/prévisualisation.
- **Actions** : **approuver** ; **rejeter + motif** (obligatoire). Confirmation + retour d'état.
- **Robustesse** : gestion 404 (dossier inconnu) et 409 (déjà décidé / re-décision) ; feedback ; rafraîchissement de la file après action (le read-model se met à jour de façon asynchrone).

**Hors périmètre :**
- Extraction OCR (assiste la revue mais backend `document-service`, plus tard) — l'humain décide (invariant DO-1).
- Octroi d'entitlement après approbation (AP-05/AP-06).
